Шаг 2. Подготовьте данные¶

Выгрузим датасет в файл Excel методом to_excel().

In [4]:
# games.to_excel('games.xlsx')

Заполним пропуски в столбце Critic_Score. Для этого воспользуемся средством автоматизации AutoIt. Напишем скрипт, который копирует название игры, платформу и год выпуска из таблицы Excel, переключается на окно браузера и вставляет значения в поисковик, допечатывая фразу "metacritic score" (сайт-агрегатор Metacritic используется в качестве основного источника оценок). На странице результатов поиска скрипт через Ctrl+f ищет первый попавшийся символ % (как правило, это и есть оценка критиков), копирует значение перед ним, переключается в окно Excel и вставляет в соответствующий столбец.

Очевидные недостатки: скорость работы — для обработки 8578 строк потребовались две ночи); ненадёжность — браузер подвисает, требуется перезапуск скрипта; неточность — может путать результат для серий игр или не выдавать ничего — для малоизвестных тайтлов. Тем не менее способ прост и обладает потенциалом для усовершенствования.

Аббревиатура tbd (To Be Determined — будет определено или To Be Decided — будет решено) в столбцах с оценками пользователей означает, что пользовательские оценки отсутствуют или их недостаточно, чтобы выставить оценку игре. Для работы со столбцом user_score необходимо избавиться от значения tbd и заменить тип данных на вещественный. Заменим значения tbd в столбце user_score на значение из столбца critic_score, поделённое на 10.

In [11]:
games2.loc[games2['user_score'] == 'tbd', 'user_score'] = games2.loc[games2['user_score'] == 'tbd', 'critic_score'] / 10

Заменим тип данных столбца user_score на вещественный методом astype().

In [12]:
games2['user_score'] = games2['user_score'].astype('float')

Заменим пропуски в столбце user_score значениями стобца critic_score, поделёнными на десять.

In [13]:
games2.loc[games2['user_score'].isnull(), 'user_score'] = games2.loc[games2['user_score'].isnull(), 'critic_score'] / 10
